# 报文转 Text

一个简单的浏览器插件，用于将报文内容快速导出为 `.txt` 文本文件。

## ✨ 功能列表

- 📝 **粘贴报文内容** — 在弹出窗口中直接粘贴或输入报文内容
- ✏️ **自定义文件名** — 支持自定义导出文件名，方便管理
- 💾 **一键导出** — 点击按钮即可将内容导出为 `.txt` 文件
- 🔄 **自动保存草稿** — 自动保存输入的文本和文件名，防止意外丢失
- ⌨️ **快捷键支持** — 使用 `Ctrl+Enter` (Mac: `Cmd+Enter`) 快速导出
- 🎨 **清晰标识** — 图标使用 "TXT" 文字，便于识别

## 实际功能说明

⚠️ **注意**：本插件当前版本是一个简化的文本导出工具，而非网络请求捕获工具。

**实际功能**：
1. 提供一个弹出窗口界面
2. 用户可以粘贴或输入任意文本内容（通常是 HTTP 报文）
3. 自定义导出文件名
4. 将内容保存为 `.txt` 文件到本地

**不适用场景**：
- 自动捕获浏览器网络请求
- 监听 HTTP/HTTPS 流量
- 自动解析报文格式

## 📦 安装教程

本插件仅支持本地文件安装到谷歌浏览器（Chrome）或 Edge 浏览器。

### 方法一：谷歌浏览器（Chrome）安装

#### 步骤 1：打开扩展程序页面

1. 打开谷歌浏览器
2. 在地址栏输入：`chrome://extensions/`
3. 按回车键进入扩展程序管理页面

#### 步骤 2：启用开发者模式

1. 在页面右上角找到**"开发者模式"**开关
2. 点击开关，将其**启用**（变为蓝色/打开状态）

#### 步骤 3：加载插件

1. 点击页面左上角的**"加载已解压的扩展程序"**按钮
2. 在弹出的文件选择窗口中，找到并选择 `network-request-saver` 文件夹
3. 点击**"选择"**（Mac）或**"确定"**（Windows）按钮

#### 步骤 4：验证安装

1. 插件安装成功后，会出现在扩展程序列表中
2. 浏览器工具栏会出现插件的图标（显示 "TXT" 文字）
3. 如果没有看到图标，点击工具栏的**拼图图标**（扩展程序菜单），找到"报文转 Text"并固定到工具栏

### 方法二：Edge 浏览器安装

#### 步骤 1：打开扩展程序页面

1. 打开 Microsoft Edge 浏览器
2. 在地址栏输入：`edge://extensions/`
3. 按回车键进入扩展程序管理页面

#### 步骤 2：启用开发者模式

1. 在页面左侧找到**"开发人员模式"**开关
2. 点击开关，将其**启用**

#### 步骤 3：加载插件

1. 点击页面上的**"加载解压缩的扩展"**按钮
2. 选择 `network-request-saver` 文件夹
3. 确认加载

### 🔄 更新插件

如果修改了插件代码或图标，需要重新加载插件：

1. 进入 `chrome://extensions/` 或 `edge://extensions/`
2. 找到"报文转 Text"插件
3. 点击**"重新加载"**按钮（刷新图标）

### ❓ 常见问题

**Q: 找不到 "加载已解压的扩展程序" 按钮？**
A: 请确保已启用"开发者模式"，只有启用后才会显示该按钮。

**Q: 安装后看不到图标？**
A: 点击浏览器工具栏的拼图图标（扩展程序菜单），找到插件并点击"固定"按钮。

**Q: 插件无法正常工作？**
A: 尝试重新加载插件，或检查浏览器控制台（右键插件图标 → 检查弹出窗口）查看错误信息。

## 📖 使用教程

### 基本使用流程

1. **打开插件**
   - 点击浏览器工具栏中的"报文转 Text"图标
   - 弹出窗口将会打开

2. **输入内容**
   - 在"报文内容"文本框中粘贴或输入内容
   - 内容可以是 HTTP 请求报文、响应报文，或任何文本内容

3. **设置文件名**
   - 在"文件名"输入框中输入自定义文件名
   - 如果留空，将自动生成文件名（格式：`request_日期时间`）

4. **导出文件**
   - 点击**"导出"**按钮
   - 选择保存位置
   - 文件将以 `.txt` 格式保存

5. **清空内容**（可选）
   - 点击**"清空"**按钮可以清除所有输入内容
   - 会弹出确认提示，防止误操作

### 快捷键

- `Ctrl+Enter` (Windows/Linux) 或 `Cmd+Enter` (Mac)：快速导出

### 自动保存

- 插件会自动保存你输入的内容和文件名（延迟 500ms）
- 关闭弹出窗口后重新打开，上次输入的内容会自动恢复
- 数据保存在浏览器本地存储中，不会上传到服务器

## 📄 导出文件格式

导出的文件是纯文本格式（`.txt`），内容为你输入的原始文本。

### 示例

如果你输入以下内容：

```
POST /api/users HTTP/1.1
Host: example.com
Content-Type: application/json

{
  "name": "John",
  "email": "john@example.com"
}
```

导出的 `.txt` 文件将完全保留这个格式和内容。

## 🔒 权限说明

插件需要以下权限：

- `downloads` — 用于将文件保存到你的电脑
- `storage` — 用于保存草稿内容（本地存储，不上传）

### 为什么需要这些权限？

- **downloads**：没有这个权限，插件无法将内容导出为文件
- **storage**：用于自动保存你输入的内容和文件名，提升用户体验

## ⚠️ 注意事项

1. **本地插件**：本插件仅通过本地文件加载，不从 Chrome Web Store 安装
2. **数据隐私**：所有数据保存在本地浏览器中，不会上传到任何服务器
3. **文件编码**：导出的文件使用 UTF-8 编码，支持中文和特殊字符
4. **文件保存位置**：默认保存到浏览器的下载目录，可在导出时选择其他位置

## 🎨 图标说明

插件图标使用 "TXT" 文字设计：

- **正常状态**：蓝色背景，白色文字
- **录制状态**：红色背景，白色文字（预留，当前版本未使用）

这种设计使得插件在工具栏中更容易识别。
